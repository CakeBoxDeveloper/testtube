# Техническое задание: MSE система (Проксима + София)

## Обзор системы

Двухмодульная система управления задачами и здоровьем с единой базой данных и двумя независимыми интерфейсами.

---

## 1. Архитектура системы

### 1.1 Компоненты

```
┌─────────────────────────────────────────┐
│         Единая база данных              │
│  (Пользователи, задачи, статы, чаты)   │
└──────────────┬──────────────────────────┘
               │
       ┌───────┴────────┐
       │                │
   ┌───▼────┐      ┌───▼────┐
   │Проксима │      │ София  │
   │(Задачи)│      │(Здоровье)
   └────────┘      └────────┘
```

### 1.2 Принцип работы

- **Единая БД**: хранит все данные пользователей
- **Модульная загрузка**: каждый модуль загружает только необходимые параметры
- **Общие параметры**: регистрация, авторизация, профиль пользователя
- **Независимые интерфейсы**: разные UI/UX для разных целей

---

## 2. Общие функции (оба модуля)

### 2.1 Регистрация и авторизация

**При регистрации:**
- Создание учётной записи (логин/пароль)
- Присвоение порядкового номера (номер звена)
- Указание возраста
- Указание пола

**Кроссмодульный доступ:**
- Пользователь, зарегистрировавшийся в Проксиме, может войти в Софию с тем же логином
- Пользователь, зарегистрировавшийся в Софии, может войти в Проксиму с тем же логином

### 2.2 Профиль пользователя

**Общие данные в БД:**
- Логин
- Пароль (хеш)
- Номер звена
- Возраст
- Пол
- Дата регистрации

---

## 3. Модуль ПРОКСИМА (Планировщик задач)

### 3.1 Назначение

Управление конкретными, краткосрочными задачами (приготовить суп, переустановить программу, починить велосипед).

### 3.2 Загружаемые из БД параметры

- Список задач пользователя
- Названия задач
- Описания задач
- Шаги выполнения
- Прогресс выполнения
- Приоритеты
- Дедлайны
- Номер звена (логин пользователя)

### 3.3 Основные компоненты UI

#### 3.3.1 Браузер задач

**Структура:**
- Общий таксклист (все задачи)
- Вкладки с конкретными пайплайнами (категории/проекты)

**Функциональность:**
- Вертикальный todo чеклист
- Отметить выполненные задачи
- Просмотр прогресса
- Фильтрация по приоритету/дедлайну

#### 3.3.2 ИИ-чат с Проксимой

**Возможности:**
- Задавать вопросы о задачах
- Просить изменить/отредактировать шаги
- Просить пояснить подробнее
- Проксима видит все задачи пользователя
- Напоминает о срочных задачах с высоким приоритетом
- Адаптирует ответы в зависимости от дедлайнов и приоритетов

#### 3.3.3 Создание задачи

**Стандартный способ:**
- Ввод названия
- Описание
- Разбор на шаги
- Установка приоритета
- Установка дедлайна

**AI Enhance (голосовой ввод):**
- Голосовой набор задачи
- Нейросеть автоматически:
  - Распознаёт речь
  - Структурирует в понятный промт
  - Разбивает на шаги
  - Предлагает приоритет и дедлайн

### 3.4 Параметры задачи

```
{
  id: UUID,
  userId: номер_звена,
  title: string,
  description: string,
  steps: [
    {
      id: UUID,
      title: string,
      completed: boolean,
      order: number
    }
  ],
  priority: "low" | "medium" | "high" | "critical",
  deadline: datetime,
  progress: number (0-100),
  category: string,
  createdAt: datetime,
  updatedAt: datetime,
  completedAt: datetime | null
}
```

## 4. Модуль СОФИЯ (Здоровье и развитие)

### 4.1 Назначение

Отслеживание здоровья, спорта, развития и обучения.

### 4.2 Загружаемые из БД параметры

- Статы пользователя (спортивные показатели)
- Номер звена
- Возраст
- Пол
- Список достижений (ачивок)
- Вес (история)
- Вкладки с чатами с моделью София
- Динамика изменений статов
- Открытые/купленные обучающие материалы

**НЕ загружает:**
- Список задач из Проксимы

### 4.3 Основные компоненты UI

#### 4.3.1 Браузер материалов

**Функциональность:**
- Скачивание обучающих материалов
- Прохождение тестов
- Использование калькуляторов
- Отслеживание открытых достижений

#### 4.3.2 ИИ-чат с Софией

**Возможности:**
- Создание новых вкладок (разные темы/направления)
- Закрепление вкладок
- Удаление вкладок
- Переименование вкладок
- Консультации по здоровью и спорту
- Анализ прогресса пользователя

#### 4.3.3 Модуль "Монитор" (адаптивный)

**Структура:**
- Три части, свайпаемые влево-вправо

**Содержимое:**
1. **Показатели здоровья**
   - Пульс
   - Давление
   - Другие биометрические данные

2. **Динамика веса**
   - График изменения веса
   - Тренд
   - Целевой вес

3. **Резюме состояния организма**
   - Общая оценка здоровья
   - Рекомендации
   - Краткий анализ

### 4.4 Адаптивная верстка

**Поведение:**
- По умолчанию: Браузер + ИИ-чат рядом
- При сворачивании одного модуля: второй растягивается на всё свободное место
- При сворачивании обоих: открывается модуль "Монитор" на весь экран

### 4.5 Параметры пользователя в Софии

```
{
  userId: номер_звена,
  stats: {
    height: number,
    weight: number,
    age: number,
    gender: "M" | "F",
    pulse: number,
    bloodPressure: string,
    // другие показатели
  },
  achievements: [
    {
      id: UUID,
      title: string,
      description: string,
      unlockedAt: datetime,
      icon: string
    }
  ],
  weightHistory: [
    {
      date: datetime,
      weight: number
    }
  ],
  chatTabs: [
    {
      id: UUID,
      title: string,
      pinned: boolean,
      messages: [...]
    }
  ],
  materials: [
    {
      id: UUID,
      title: string,
      type: "course" | "test" | "calculator",
      purchased: boolean,
      progress: number
    }
  ]
}
```

## 5. Взаимодействие модулей

### 5.1 Общие данные

| Параметр | Проксима | София |
|----------|----------|-------|
| Логин | ✓ | ✓ |
| Пароль | ✓ | ✓ |
| Номер звена | ✓ | ✓ |
| Возраст | ✓ | ✓ |
| Пол | ✓ | ✓ |
| Задачи | ✓ | ✗ |
| Статы здоровья | ✗ | ✓ |
| Достижения | ✗ | ✓ |
| Вес | ✗ | ✓ |

### 5.2 Независимость модулей

- Каждый модуль имеет собственный интерфейс
- Каждый модуль загружает только нужные данные
- Изменения в одном модуле не влияют на другой (кроме общих параметров профиля)
- Возможна работа в одном модуле без использования другого

## 6. Требования к БД

### 6.1 Таблицы

1. **users**
   - id (UUID)
   - login (string, unique)
   - password (string, hashed)
   - звено_номер (int, auto-increment)
   - age (int)
   - gender (enum: M/F)
   - created_at (datetime)
   - updated_at (datetime)

2. **tasks**
   - id (UUID)
   - user_id (FK → users.id)
   - title (string)
   - description (text)
   - priority (enum: low/medium/high/critical)
   - deadline (datetime)
   - progress (int 0-100)
   - category (string)
   - created_at (datetime)
   - updated_at (datetime)
   - completed_at (datetime, nullable)

3. **task_steps**
   - id (UUID)
   - task_id (FK → tasks.id)
   - title (string)
   - completed (boolean)
   - order (int)
   - created_at (datetime)

4. **user_stats**
   - id (UUID)
   - user_id (FK → users.id)
   - height (float)
   - weight (float)
   - pulse (int)
   - blood_pressure (string)
   - updated_at (datetime)

5. **weight_history**
   - id (UUID)
   - user_id (FK → users.id)
   - weight (float)
   - recorded_at (datetime)

6. **achievements**
   - id (UUID)
   - user_id (FK → users.id)
   - title (string)
   - description (text)
   - unlocked_at (datetime)
   - icon (string)

7. **chat_tabs**
   - id (UUID)
   - user_id (FK → users.id)
   - module (enum: proxima/sofia)
   - title (string)
   - pinned (boolean)
   - created_at (datetime)

8. **chat_messages**
   - id (UUID)
   - tab_id (FK → chat_tabs.id)
   - sender (enum: user/ai)
   - content (text)
   - created_at (datetime)

9. **materials**
   - id (UUID)
   - title (string)
   - type (enum: course/test/calculator)
   - content (text/blob)
   - created_at (datetime)

10. **user_materials**
    - id (UUID)
    - user_id (FK → users.id)
    - material_id (FK → materials.id)
    - purchased (boolean)
    - progress (int 0-100)
    - accessed_at (datetime)

## 7. Требования к ИИ-моделям

### 7.1 Проксима (AI для задач)

**Возможности:**
- Распознавание речи (для AI Enhance)
- Структурирование задач
- Разбор на шаги
- Анализ приоритета и дедлайна
- Контекстные напоминания
- Редактирование и уточнение задач

**Контекст:**
- Видит все задачи пользователя
- Знает приоритеты и дедлайны
- Адаптирует тон в зависимости от срочности

### 7.2 София (AI для здоровья)

**Возможности:**
- Консультации по здоровью и спорту
- Анализ статов пользователя
- Рекомендации по улучшению
- Отслеживание прогресса
- Мотивирующие сообщения

**Контекст:**
- Видит статы пользователя
- Знает историю веса
- Учитывает возраст и пол
- Анализирует динамику изменений

## 8. Требования к безопасности

- Хеширование паролей (bcrypt/argon2)
- JWT токены для сессий
- HTTPS для всех запросов
- Валидация входных данных
- Защита от SQL-инъекций
- CORS для кроссмодульного доступа

---

## 9. Требования к производительности

- Загрузка модуля < 2 сек
- Ответ ИИ-чата < 5 сек
- Кэширование часто используемых данных
- Оптимизация запросов к БД
- Ленивая загрузка материалов

---

## 10. Требования к UI/UX

### 10.1 Адаптивность

- Мобильная версия (< 768px)

- Клавиатурная навигация
